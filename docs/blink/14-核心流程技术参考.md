### 1.4 JS 编译

**文件**: `crates/toolchain/src/js.rs`

- `compile_js(js, element_vars, css_rules, builtins) -> (Vec<EventHandler>, Vec<SharedStateVar>)` `:1635`

### 4.1 Dirty 标记系统

**文件**: `crates/dom/src/node.rs`

```

  is_text_only_change() -> bool :583  脏子树是否为纯文本变更?
```

### 4.2 文本原地更新 (避免 Rc 替换)

**文件**: `crates/dom/src/node.rs`

```
Node::update_text_content(&mut self, text)  :296
  ├─ 若首个子节点是 Text:
  │   ├─ Text::set_data(text)      原地修改字符串，不分配新 Rc
  │   ├─ child.dirty.set(true)     在 Text 子节点上标脏
  │   └─ 手动沿 self.parent 链传播 has_dirty_child
  │       (避免 mark_dirty 的 parent 回借 RefCell panic)
  └─ 否则回退到 set_text_content()  :290 (创建新 Text 节点，触发全量重建)
```

**为什么不能直接调用 `mark_dirty`**: `update_text_content` 持有 `&mut self`（元素自身），若从 Text 子节点调用 `mark_dirty`，传播第一步就是 `parent.borrow()`（parent 正是 self），导致 RefCell 双重借用 panic。

### 4.3 增量路径判断

**文件**: `crates/renderer/src/window.rs`

```
App::relayout()  :412
  ├─ 计算 viewport, 检查 size_changed
  ├─ body.has_any_dirty()  → 无脏标记且非 size 变更 → 直接返回
  ├─ **增量路径**: !size_changed && body.is_text_only_change()
  │   ├─ clear_dirty()
  │   └─ DisplayListBuilder::build(&existing_layout_root)
  │       从**现有**布局树构建 DisplayList（读取最新 DOM 文本）
  │       **跳过**: compute_dom_styles / build_layout_tree / layout_engine.layout()
  │
  └─ **全量路径**: size_changed || 结构变更
      ├─ compute_dom_styles()        全量样式重算
      ├─ build_layout_tree()         重建布局树
      ├─ layout_engine.layout()      全量布局计算
      └─ DisplayListBuilder::build() 重建 DisplayList
```

### 4.4 增量 vs 全量边界

| 触发事件 | dirty 标记位置 | 走哪条路径 |
|---------|---------------|-----------|
| 键盘输入 (Backspace/字符) | Text 子节点 | **增量**: 跳过样式+布局 |
| 鼠标点击 (可能 DOM 变更) | 结构变更则元素自身 dirty | **全量** (如有 append_child 等) |
| 窗口 resize | — (size_changed=true) | **全量** |
| 仅点击无 DOM 变更 | 无 | 直接返回 (no-op) |
| `set_text_content()` | 元素自身 dirty | **全量** (Rc 被替换) |

**字体缓存**: `crates/renderer/src/text_renderer.rs`

- `GlyphCache` 结构体 `:21` — 缓存键: (font_id, glyph_id, font_size_px), 值: 图集 UV 坐标 + 尺寸
- 缓存未命中时走光栅化路径，命中时直接复用图集位置

### 5.4 字体相关数据流

```
PaintCommand::Text { text, font_size, font_family, font_weight, x, y, color }
  → TextRenderer::prepare_text()
    → fontdb 查找: family + weight → font_id
    → rustybuzz 塑形: font_id + text + size → [GlyphInfo]
    → 字形光栅化: ttf_parser + ab_glyph → 位图
    → 写入纹理图集: write_texture()
    → 输出实例数据: [x, y, w, h, u, v, u2, v2, r, g, b, a]
  → GPU text pass:
    → 顶点着色器: 实例数据 → 屏幕坐标四边形
    → 片段着色器: 纹理采样 + color tint
```

### 6.3 滚动 (当前状态: 未实现)

滚动条/overflow:scroll 在当前 Phase 0 尚未实现。相关基础设施已就位:

- `LayoutBox.overflow: Overflow` (layout_box.rs:131) — 解析自 CSS overflow
- `is_bfc_container()` (block.rs:113) — overflow != visible 的容器创建 BFC
- `WgpuBackend::push_clip(rect)` / `pop_clip()` (wgpu_backend.rs:901-905) — 裁剪栈

滚动实现路线 (Phase 2+):
1. 鼠标滚轮事件 → 调整容器的 scroll_offset
2. 视口裁剪 (P0-5 `is_visible_in_viewport`) 需要处理带 scroll_offset 的坐标变换
3. 渲染时应用 `translate(-scroll_offset.x, -scroll_offset.y)`
4. 裁剪区域 = 容器的 border-box
