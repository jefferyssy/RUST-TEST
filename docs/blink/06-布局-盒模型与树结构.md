# 06 — 布局盒模型与树结构：enum + Vec 所有权 vs 继承体系 + GC 托管

## 问题/目标

布局树（Layout Tree / Render Tree）是 DOM 树经过 CSS 级联后的"平面化"表示。它与 DOM 树形状相似但不完全相同：
- 某些 DOM 节点不产生布局框（`display: none`、注释节点）
- 某些 DOM 节点产生多个布局框（如行内元素跨行分裂）
- 匿名框：为满足 CSS 盒模型约束而自动插入的不可见框（如 Flex/Grid 容器内的匿名项）

关键约束：
- 布局树需要从 DOM 树构建，同时需要 CSS ComputedStyle 作为输入
- 布局算法需要遍历树并修改节点位置和尺寸
- 布局树与 DOM 树保持双向引用以便增量更新

## 我们的实现

### LayoutBox：枚举分发 + 数据集中

来源：[crates/layout/src/layout_box.rs](../../crates/layout/src/layout_box.rs)

```rust
/// 布局框类型 —— 参与布局计算的基本单元
#[derive(Debug, Clone, PartialEq)]
pub enum BoxType {
    Block,
    Inline,
    FlexContainer,
    FlexItem,
    Text,
    Anonymous,
    // Phase 1 新增
    InlineBlock,
    Table, TableRow, TableCell,
    Absolute, Fixed, Sticky,
    // Phase 2 新增
    GridContainer, GridItem,
    Float,
}

/// 布局树节点
pub struct LayoutBox {
    pub box_type: BoxType,
    pub node: Option<Rc<RefCell<Node>>>,   // ★ 回指 DOM 节点
    pub children: Vec<LayoutBox>,           // ★ 所有权在父节点
    pub rect: Rect<f32>,                    // 计算结果
    pub padding: EdgeSizes,
    pub margin: EdgeSizes,
    pub border: EdgeSizes,
    pub computed_style: Option<ComputedStyle>,
    // Phase 1+ 扩展
    pub z_index: i32,
    pub stacking_context: bool,
    pub overflow: Overflow,
    pub border_radius: BorderRadius,
    pub visibility: Visibility,
    pub dirty: bool,                        // 增量布局脏标记
}
```

**关键设计决策**：

1. **单一 enum 表达所有框类型**：15 个变体，一个 match 即可分发到不同算法
2. **`Vec<LayoutBox>` 所有权**：父节点拥有子节点，无需 GC 或引用计数
3. **`node: Option<Rc<RefCell<Node>>>`**：轻量回指 DOM，用于增量更新时查找
4. **所有盒模型数据扁平存储**：`padding`/`margin`/`border`/`rect` 等字段直接在 `LayoutBox` 上，不分层

### 构建布局树

来源：[crates/layout/src/lib.rs](../../crates/layout/src/lib.rs#L378)

```rust
pub fn build_layout_tree(
    dom_root: &Rc<RefCell<Node>>,
    computed_styles: &HashMap<usize, ComputedStyle>,
    text_measurer: Option<&mut TextMeasurer>,
) -> LayoutBox {
    build_layout_box(dom_root, computed_styles, 0, text_measurer)
}

fn build_layout_box(
    dom_node: &Rc<RefCell<Node>>,
    computed_styles: &HashMap<usize, ComputedStyle>,
    _depth: usize,
    mut text_measurer: Option<&mut TextMeasurer>,
) -> LayoutBox {
    let node = dom_node.borrow();
    let ptr = Rc::as_ptr(dom_node) as usize;
    let style = computed_styles.get(&ptr);

    match &node.node_type {
        NodeType::Element(elem_data) => {
            // 1. 从 CSS display 属性判断 BoxType
            let box_type = match style.and_then(|s| s.get("display")) {
                Some(v) if format!("{:?}", v).contains("grid") => BoxType::GridContainer,
                Some(v) if format!("{:?}", v).contains("flex")  => BoxType::FlexContainer,
                Some(v) if format!("{:?}", v).contains("inline-block") => BoxType::InlineBlock,
                Some(v) if format!("{:?}", v).contains("inline") => BoxType::Inline,
                // ...
                _ => BoxType::Block,
            };

            // 2. 应用 UA 默认样式（按 HTML 标签名）
            let box_type = if display.is_empty() && box_type == BoxType::Block {
                match elem_data.tag_name() {
                    "span" | "a" | "em" | "strong" | "img" | ... => BoxType::Inline,
                    "button" | "input" | ... => BoxType::InlineBlock,
                    _ => BoxType::Block,
                }
            } else { box_type };

            // 3. position/filter 覆盖类型
            // position: absolute → BoxType::Absolute
            // float: left → BoxType::Float

            let mut layout_box = LayoutBox::new(box_type, Some(dom_node.clone()));
            layout_box.computed_style = style.cloned();

            // 4. 解析盒模型（padding/margin/border）
            apply_shorthand_property(style, &mut layout_box);

            // 5. 递归构建子节点
            drop(node);
            for child in &dom_node.borrow().child_nodes() {
                let child_box = build_layout_box(child, computed_styles, _depth + 1, ...);
                layout_box.append_child(child_box);
            }

            // 6. 初始尺寸估算（最大子节点宽度 + padding）
            // 后续 layout() 阶段会修正
            layout_box
        }
        NodeType::Text(_) => {
            // 文本节点：估算尺寸（font-size * char_width 或 rustybuzz 精确测量）
            let mut layout_box = LayoutBox::new(BoxType::Text, Some(dom_node.clone()));
            layout_box.rect.height = font_size * 1.2;
            layout_box.rect.width = measurer.measure_width(text, font_size, ...);
            layout_box
        }
        NodeType::Comment(_) => {
            // 注释节点不参与布局
            LayoutBox::new(BoxType::Anonymous, Some(dom_node.clone()))
        }
    }
}
```

**构建特点**：
- 全量重建：每次 DOM 变更 → 整个布局树从根重建
- 初始估算宽度 = max(子节点宽度) + padding，高度 = sum(子节点高度)
- Text 节点在构建时即估算尺寸（rustybuzz 精确测量或字符宽度估算）

```rust
fn calculate_sizes(&mut self, node: &mut LayoutBox, viewport: Size<f32>, parent_box_type: Option<BoxType>) {
    match node.box_type {
        BoxType::FlexContainer => {
            self.calculate_sizes_children(node, viewport);   // 自底向上先算子级
            FlexLayout.layout(&mut self.taffy, node, viewport, constrain_width);
        }
        BoxType::Block | BoxType::Anonymous => {
            BlockLayout.layout(node, viewport);               // 逐行 Y 堆叠
        }
        BoxType::Absolute | BoxType::Fixed => {
            BlockLayout.layout(node, viewport);               // 先按块级计算
            // PositionedLayout 在 layout() 最后统一处理偏移
        }
        BoxType::GridContainer => {
            GridLayout.layout(node, viewport);
        }
        // ...
    }
    self.calculate_sizes_children(node, viewport);           // 递归子节点
    self.shrink_to_content(node);                            // 收缩包裹高度
}
```

## Blink 的实现

### LayoutObject 继承体系

Blink 的布局树节点采用**深度继承层次**，每个子类封装特定布局行为：

```cpp
// ===== 核心继承链（core/layout/）=====

// 基类：所有布局对象的公共基类
class LayoutObject : public GarbageCollected<LayoutObject> {
public:
    // 树结构
    LayoutObject* Parent() const;
    LayoutObject* NextSibling() const;
    LayoutObject* PreviousSibling() const;
    LayoutObject* FirstChild() const;
    LayoutObject* LastChild() const;

    // 回指 DOM
    Node* GetNode() const;                          // 永远非空
    Node* NonPseudoNode() const;                    // 跳过伪元素

    // 几何信息（相对于包含块）
    PhysicalRect FrameRect() const;                 // border-box

    // 虚方法：各子类重写以提供特定行为
    virtual bool IsOfType(LayoutObjectType) const;  // 类型检查（位掩码）
    virtual void UpdateLayout();                     // 重新布局
    virtual const char* GetName() const;            // 调试名称
    virtual void AddChild(LayoutObject* new_child, LayoutObject* before_child = nullptr);
    virtual void RemoveChild(LayoutObject* old_child);

    // 脏标记
    bool NeedsLayout() const;
    void SetNeedsLayout(LayoutInvalidationReason, MarkingBehavior);

    // 样式引用（不可变 + CoW）
    const ComputedStyle& StyleRef() const;
    const ComputedStyle* Style() const;
    void SetStyle(const ComputedStyle*, StyleDifference*);

protected:
    Member<LayoutObject> parent_;
    Member<LayoutObject> previous_;
    Member<LayoutObject> next_;
    Member<LayoutObject> first_child_;
    Member<LayoutObject> last_child_;
    Member<const ComputedStyle> style_;             // ★ 不可变 ComputedStyle 引用
};

// 抽象中间层
class LayoutBox : public LayoutObject {
public:
    PhysicalRect FrameRect() const;
    LayoutUnit X() const, Y() const;
    LayoutUnit LogicalWidth() const;
    LayoutUnit LogicalHeight() const;

    // 盒模型（由子类 LayoutBoxModelObject 提供）
    virtual LayoutUnit PaddingTop() const;
    virtual LayoutUnit BorderTop() const;
};

// 盒模型层：具有 CSS 盒模型的元素
class LayoutBoxModelObject : public LayoutBox {
public:
    // 从 ComputedStyle 实时计算盒模型值
    LayoutUnit PaddingTop() const override;
    LayoutUnit PaddingRight() const override;
    LayoutUnit BorderTop() const override;
    LayoutUnit MarginTop() const override;
    // ...

    // 绘制层背景
    PhysicalRect PhysicalBackgroundRect() const;
    // ...
};

// 块级布局
class LayoutBlock : public LayoutBoxModelObject {
public:
    // 块级格式化上下文相关
    bool CreatesNewFormattingContext() const;
    LayoutUnit TextIndentOffset() const;

    // BFC 内布局入口
    void LayoutBlock(bool relayout_children);
};

// 具体块级实现（最常见的布局对象）
class LayoutBlockFlow : public LayoutBlock {
public:
    void UpdateLayout() override;  // ← LayoutNG 入口
    void AddChild(LayoutObject*, LayoutObject* before) override;

    // LayoutNG 集成
    const NGPhysicalBoxFragment* CurrentFragment() const;
};

// Flex 容器
class LayoutNGFlexibleBox : public LayoutBlock {
public:
    void UpdateLayout() override;
};

// Grid 容器
class LayoutNGGrid : public LayoutBlock {
public:
    void UpdateLayout() override;
};

// 行内元素
class LayoutInline : public LayoutBoxModelObject { ... };

// 文本
class LayoutText : public LayoutObject {
public:
    StringView Text() const;
    // 文本不使用盒模型，直接继承 LayoutObject
};

// 定位元素层
class LayoutNGFieldset : public LayoutBlockFlow { ... };
class LayoutTable : public LayoutBlock { ... };
class LayoutTableRow : public LayoutBox { ... };     // 不继承 Block（不是块级）
class LayoutTableCell : public LayoutBlockFlow { ... };
```

**继承层次深度**：`LayoutText → LayoutObject → LayoutBox → LayoutBoxModelObject → LayoutBlock → LayoutBlockFlow`（6 层）

**Rust 对比**：我们的 `BoxType` enum 与 Blink 的 `LayoutObjectType` 位掩码等价，但 `match` 天然穷尽检查，比位掩码更安全。

### 盒模型值：从 ComputedStyle 实时计算

```cpp
// Blink 的盒模型值不存储在 LayoutObject 中，而是从 ComputedStyle 实时计算
class LayoutBoxModelObject : public LayoutBox {
    LayoutUnit PaddingTop() const override {
        return StyleRef().PaddingTop().ConvertToLength(resolver);    // ← CoW ComputedStyle
    }
    LayoutUnit BorderTop() const override {
        return StyleRef().BorderTop().width;                          // ← 直接取字段
    }
    LayoutUnit MarginTop() const override {
        return StyleRef().MarginTop().ConvertToLength(resolver);
    }
};
```

这与我们不同：我们将 `padding`/`margin`/`border` 预解析为 `EdgeSizes` 存储在 `LayoutBox` 中；Blink 每次访问时从 `ComputedStyle` 实时计算（得益于 CoW 共享，开销极低）。

### 布局树构建流程

```cpp
// StyleResolver 负责样式更新 + 布局树构建
void StyleResolver::UpdateStyleAndLayoutTree() {
    // 只处理脏节点
    for (Element* element : dirty_elements_) {
        // 1. 重新计算 ComputedStyle（CoW 共享大多数组）
        const ComputedStyle* new_style = ResolveStyle(element, parent_style);
        element.SetComputedStyle(new_style);

        // 2. 附加/更新/移除对应的 LayoutObject
        if (element.GetLayoutObject()) {
            element.GetLayoutObject()->SetStyle(new_style, &diff);
        } else if (NeedsLayoutObject(new_style)) {
            // 为元素创建正确的 LayoutObject 子类
            LayoutObject* new_layout_object = CreateLayoutObject(element, new_style);
            element.SetLayoutObject(new_layout_object);
            parent_layout_object->AddChild(new_layout_object);
        }
    }
}

// LayoutObject 工厂 —— 按 display 值创建具体子类
LayoutObject* CreateLayoutObject(Element* element, const ComputedStyle& style) {
    switch (style.Display()) {
        case EDisplay::kBlock:      return MakeGarbageCollected<LayoutBlockFlow>(element);
        case EDisplay::kFlex:       return MakeGarbageCollected<LayoutNGFlexibleBox>(element);
        case EDisplay::kGrid:       return MakeGarbageCollected<LayoutNGGrid>(element);
        case EDisplay::kInline:     return MakeGarbageCollected<LayoutInline>(element);
        case EDisplay::kTable:      return MakeGarbageCollected<LayoutTable>(element);
        case EDisplay::kTableCell:  return MakeGarbageCollected<LayoutTableCell>(element);
        case EDisplay::kNone:       return nullptr;  // ★ 不创建布局对象
        // ...
    }
}
```

```cpp
        }
        LayoutBlock::AddChild(new_child, before_child);
    }
```

## 优劣势分析

| 维度 | 我们 (enum + Vec) | Blink (继承体系) |
|------|------------------|------------------|
| **类型安全** | `match` 穷尽检查，编译期保证 | `IsOfType()` 运行时检查 + `switch` |
| **多态行为** | 外部 match 分发 | 虚函数动态分发，OCP 开放性好 |
| **代码组织** | 同一个函数处理所有类型（集中） | 每个子类封装自身行为（分散） |
| **盒模型存储** | 预解析 `EdgeSizes` 在 `LayoutBox` 上 | 实时从 ComputedStyle 计算（CoW 低成本） |
| **构建方式** | 全量递归重建（O(N) ~200μs/node） | 增量更新（O(脏节点数)） |
| **树所有权** | `Vec<LayoutBox>` 父拥有子 | GC 托管，无所有权概念 |
| **匿名框** | `BoxType::Anonymous` + 手动插入 | 各子类 `AddChild` 自动插入 |
| **内存占用** | ~160B/node（扁平 struct） | ~200B+/node（虚表指针 + GC header + 更多字段） |
| **扩展成本** | 添加 enum 变体 + 每个 match 加分支 | 添加新子类，虚函数可选重写 |
| **匿名文本框** | Text 节点直接作为叶子 | 多层级展平（LayoutText 直接挂在 LayoutBlock 下） |

### 我们的优势

1. **数据局部性好**：`LayoutBox` 是扁平 struct，所有字段连续内存，缓存友好
2. **所有权清晰**：父节点 `Vec<LayoutBox>` 持有子节点，无需考虑生命周期
3. **模式匹配安全**：`match box_type` 编译期穷尽检查，不会遗漏类型
4. **无虚表开销**：无需 vtable 指针，每个节点省 8 bytes
5. **序列化天然**：`#[derive(Debug, Clone)]` 即可，无继承序列化问题

### 我们的劣势

1. **全量重建**：每次 DOM 变更重建整个布局树，O(树大小) 而非 O(脏节点数)
2. **match 蔓延**：每个新 BoxType 变体需要更新所有 match 分支（分散在多文件）
3. **盒模型冗余**：`padding`/`margin`/`border` 预解析存储在 LayoutBox 中，与 ComputedStyle 重复
4. **无虚函数扩展点**：第三方无法添加新 BoxType 并注入布局行为
5. **匿名框需要手动**：Flex/Grid 容器内匿名项包装需要布局算法自行处理
6. **Text 尺寸估算**：构建阶段估算，非精确值，需后续 layout() 修正

### Blink 的优势

1. **增量构建 + 更新**：只重建脏节点的 LayoutObject，样式变更不重建布局树
2. **虚函数定制**：每个子类重写 `UpdateLayout()`，行为内聚
3. **按需计算盒模型**：padding 等值从 ComputedStyle 实时取，无冗余
4. **成熟的匿名框机制**：`AddChild` 自动插入，布局算法假设已满足约束
5. **与 LayoutNG 深度集成**：LayoutBlockFlow → NGBlockLayoutAlgorithm → NGFragment pipeline
6. **位掩码高效类别判断**：`IsTablePart()` 一次位运算检查多类型

### Blink 的劣势

1. **继承层次深**：6 层继承链，理解和调试成本高
2. **虚表开销**：每个对象有 vtable 指针
3. **GC 耦合**：LayoutObject 必须继承 `GarbageCollected`，不能脱离 GC 使用
4. **接口复杂**：LayoutBoxModelObject 有 100+ 方法
5. **类型检查运行时**：`static_cast` / `dynamic_cast` 有成本，类型错误运行时才发现

## 改进方向

### 短期

1. **用 `&ComputedStyle` 替代预解析 EdgeSizes**：`padding()`/`margin()` 方法改为从 `computed_style` 实时取值，消除冗余存储，样式变更时无需同步更新
2. **拆分 BoxType 为 trait**：用 trait 封装布局行为，减少 match 分支（如 `trait LayoutAlgorithm { fn layout(&self, ...) }`）

### 中期

3. **增量构建布局树**：配合 dirty 标记，只重建 DOM 变更节点的子树，而非全树重建
   ```rust
   fn update_layout_tree(root: &mut LayoutBox, dirty_dom: &[Rc<RefCell<Node>>]) {
       for dirty in dirty_dom {
           if let Some(layout_node) = root.find_layout_node(dirty) {
               rebuild_subtree(layout_node);  // 只重建脏子树
           }
       }
   }
   ```
4. **匿名框自动化**：在 `append_child` 时自动插入匿名框，而非在各算法中处理

### 长期

5. **LayoutBox trait + 子类型**：用 trait object 替代 enum，每个 BoxType 成为独立类型
   ```rust
   trait LayoutObject {
       fn box_type(&self) -> BoxType;
       fn rect(&self) -> Rect<f32>;
       fn layout(&mut self, ctx: &LayoutContext);
   }
   struct BlockBox { common: LayoutCommon, ... }
   struct FlexBox { common: LayoutCommon, ... }
   ```
6. **Fragment 缓存**：参考 LayoutNG，布局结果存为不可变 `Fragment`，布局输入不变时直接复用
