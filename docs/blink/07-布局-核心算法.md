# 07 — 布局核心算法：手写分发 + taffy vs LayoutNG 算法族

## 问题/目标

布局引擎的核心任务是：给定布局树和视口尺寸，计算每个元素的精确位置和尺寸。CSS 布局规范定义了多种布局模式：
- **Block Layout**（CSS 2.1 §9）：块级元素从上到下堆叠，含外边距折叠
- **Inline Layout**（CSS 2.1 §10）：行内元素从左到右排列，含换行
- **Flexbox**（CSS Flexbox Level 1）：一维弹性布局，主轴/交叉轴对齐
- **Grid**（CSS Grid Level 1）：二维网格布局，轨道定义 + 区域定位
- **Positioned**（CSS 2.1 §9.3）：absolute/fixed/sticky 脱离正常流
- **Float**（CSS 2.1 §9.5）：浮动元素，文字环绕
- **Table**（CSS 2.1 §17）：表格布局

关键约束：
- 子元素尺寸可能依赖父容器（如 `width: 50%`），父容器高度可能依赖子元素（收缩包裹）
- 不同布局模式间存在交互（如 Flex 容器内的绝对定位子元素）
- 需要支持增量重排，不能每次都全量计算

## 我们的实现

### 算法分发架构

来源：[crates/layout/src/lib.rs](../../crates/layout/src/lib.rs#L65)

```rust
impl LayoutEngine {
    pub fn layout(&mut self, root: &mut LayoutBox, viewport: Size<f32>) {
        root.rect.width = viewport.width;
        self.calculate_sizes(root, viewport, None);     // 主递归
        if root.rect.height < viewport.height {
            root.rect.height = viewport.height;          // 撑满视口
        }
        PositionedLayout.layout(root, viewport);         // 后置定位
    }

    fn calculate_sizes(&mut self, node: &mut LayoutBox, viewport: Size<f32>, parent_box_type: Option<BoxType>) {
        match node.box_type {
            BoxType::FlexContainer => {
                self.calculate_sizes_children(node, viewport);  // ★ 自底向上：先算子级
                FlexLayout.layout(&mut self.taffy, node, viewport, constrain_width);
            }
            BoxType::Block | BoxType::Anonymous => {
                BlockLayout.layout(node, viewport);              // ★ 自顶向下
            }
            BoxType::GridContainer => {
                GridLayout.layout(node, viewport);
            }
            BoxType::Absolute | BoxType::Fixed => {
                BlockLayout.layout(node, viewport);              // 先按块级占位
            }
            // ... 其他类型
        }
        self.apply_aspect_ratio(node);
        self.calculate_sizes_children(node, viewport);           // ★ 递归子节点
        self.shrink_to_content(node);                            // ★ 收缩包裹

        // ★ 子节点布局后重跑 Block（兄弟节点位置可能已变）
        match node.box_type {
            BoxType::Block | BoxType::Anonymous | BoxType::Inline | BoxType::InlineBlock => {
                BlockLayout.layout(node, viewport);
            }
            _ => {}
        }
    }
}
```

**执行顺序特点**：
1. FlexContainer 先算子级（自底向上），再跑 taffy
2. Block 先排当前层，再算子级（自顶向下）
3. 所有节点在子级之后 shrink_to_content
4. Block 容器在子级布局后**重跑一次**（处理兄弟节点位置漂移，如 footer 跟随动态列表）

### BlockLayout：Y 轴堆叠 + 外边距折叠

来源：[crates/layout/src/block.rs](../../crates/layout/src/block.rs)

```rust
impl BlockLayout {
    pub fn layout(&self, container: &mut LayoutBox, _viewport: Size<f32>) {
        let content_width = container.content_area().width;
        let mut current_y = container.rect.y;
        let mut prev_margin_bottom = 0.0;

        let text_align = container.computed_style
            .and_then(|s| s.get("text-align"))
            .map(|v| match v { CSSValue::Keyword(k) => k.as_ref(), _ => "left" })
            .unwrap_or("left");

        for child in &mut container.children {
            // 1. 外边距折叠
            let collapsed_offset = Self::collapse_margins(prev_margin_bottom, child.margin.top);
            current_y += collapsed_offset;

            // 2. X 定位（含 padding/border 偏移）
            child.rect.x = container.rect.x + container.padding.left + container.border.left;

            // 3. 宽度约束
            match child.box_type {
                BoxType::Block | BoxType::Anonymous | BoxType::FlexContainer
                | BoxType::GridContainer | BoxType::Table => {
                    child.rect.width = content_width;           // ★ 撑满内容区
                }
                BoxType::Inline | BoxType::InlineBlock | BoxType::Text
                | BoxType::FlexItem | BoxType::GridItem => {
                    // ★ 收缩到内容宽度 + text-align 对齐
                    let available = content_width - child.rect.width;
                    match text_align {
                        "center" => child.rect.x += (available / 2.0).max(0.0),
                        "right" | "end" => child.rect.x += available.max(0.0),
                        _ => {}
                    }
                }
                _ => {}
            }

            // 4. Y 定位
            child.rect.y = current_y + container.padding.top + container.border.top;
            current_y += child.rect.height;
            prev_margin_bottom = child.margin.bottom;
        }
    }

    /// CSS 2.1 §8.3.1: 相邻块级元素垂直 margin 取较大值
    fn collapse_margins(prev_bottom: f32, current_top: f32) -> f32 {
        if prev_bottom >= 0.0 && current_top >= 0.0 {
            prev_bottom.max(current_top)                      // ★ 正 margin：取大
        } else if prev_bottom <= 0.0 && current_top <= 0.0 {
            prev_bottom.min(current_top)                      // ★ 负 margin：取小
        } else {
            prev_bottom + current_top                         // ★ 正负混合：相加
        }
    }
}
```

**支持的功能**：
- 块级元素撑满父容器宽度 ✓
- 行内/文本元素收缩包裹 + text-align 对齐 ✓
- 相邻块级 margin 折叠 ✓
- width/max-width/min-width 约束 ✓
- `margin: 0 auto` 水平居中 ✓

**不支持**：
- BFC（Block Formatting Context）隔离
- `clear: left/right/both`
- 浮动元素环绕
- `overflow: hidden` 触发 BFC

### FlexLayout：委托 taffy

来源：[crates/layout/src/flex.rs](../../crates/layout/src/flex.rs)

```rust
impl FlexLayout {
    pub fn layout(&mut self, taffy: &mut Option<TaffyTree>, container: &mut LayoutBox,
                  viewport: Size<f32>, constrain_width: bool) {
        let taffy_tree = taffy.as_mut().unwrap();

        // 1. 创建 taffy 节点
        let container_style = Self::convert_container_style(container, viewport, constrain_width);
        let container_id = taffy_tree.new_leaf(container_style).unwrap();

        for child in &container.children {
            let child_style = Self::convert_child_style(child, viewport);
            let child_id = taffy_tree.new_leaf(child_style).unwrap();
            taffy_tree.add_child(container_id, child_id).unwrap();
        }

        // 2. 调用 taffy 计算
        let available = taffy::Size {
            width:  if constrain_width { Definite(container.rect.width) } else { MaxContent },
            height: MaxContent,
        };
        taffy_tree.compute_layout(container_id, available).unwrap();

        // 3. 写回结果
        container.rect.height = taffy_tree.layout(container_id).size.height;
        for (i, child) in container.children.iter_mut().enumerate() {
            let layout = taffy_tree.layout(child_id);
            child.rect.x = container_x + layout.location.x;
            child.rect.y = container_y + layout.location.y;
            child.rect.width = layout.size.width;
            child.rect.height = layout.size.height;
        }

        taffy_tree.clear();  // ★ 为下次布局重置
    }
}
```

**taffy 属性映射**：
| CSS 属性 | taffy 字段 | 说明 |
|----------|-----------|------|
| `flex-direction` | `style.flex_direction` | row/column/row-reverse/column-reverse |
| `justify-content` | `style.justify_content` | flex-start/center/space-between/space-around/space-evenly |
| `align-items` | `style.align_items` | flex-start/center/stretch/baseline |
| `gap` | `style.gap` | 行/列间距 |
| `flex-wrap` | `style.flex_wrap` | wrap/wrap-reverse/no-wrap |
| `flex-grow/shrink/basis` | `style.flex_grow/shrink/basis` | 弹性系数 |
| `padding/border` | `style.padding/border` | 盒模型 |

**预计算策略**：Flex 容器的子节点在进 taffy 之前已通过 `calculate_sizes_children` 完成预计算，`rect.width/height` 作为 taffy 的 `min_size` 传入，防止塌陷为零。

### PositionedLayout：偏移解析

来源：[crates/layout/src/positioned.rs](../../crates/layout/src/positioned.rs)

```rust
impl PositionedLayout {
    fn layout_absolute(&self, node: &mut LayoutBox, _viewport: Size<f32>,
                       ancestor: &Option<PositionedAncestorInfo>) {
        let (top, right, bottom, left) = Self::parse_offsets(node);
        if let Some(anc) = ancestor {
            // 相对已定位祖先的内容区计算位置
            if left != 0.0  { node.rect.x = anc.rect.x + anc.padding_left + left; }
            if top != 0.0   { node.rect.y = anc.rect.y + anc.padding_top + top; }
            // right/bottom 类似，从右/下边缘反向计算
        }
    }

    fn layout_fixed(&self, node: &mut LayoutBox, viewport: Size<f32>) {
        let (top, right, bottom, left) = Self::parse_offsets(node);
        // 相对视口定位
        if left != 0.0  { node.rect.x = left; }
        if top != 0.0   { node.rect.y = top; }
    }

    fn apply_relative_offset(&self, node: &mut LayoutBox) {
        // position: relative — 从正常流位置偏移
        node.rect.x += x_offset;
        node.rect.y += y_offset;
    }
}
```

**定位祖先传递**：通过 `PositionedAncestorInfo` 值类型向下传递（避免借用冲突）。

### GridLayout：轨道解析 + 均匀分配

来源：[crates/layout/src/grid.rs](../../crates/layout/src/grid.rs)

```rust
impl GridLayout {
    pub fn layout(&self, container: &mut LayoutBox, _viewport: Size<f32>) {
        let (col_tracks, row_tracks) = Self::parse_grid_template(container);
        let col_widths = Self::resolve_track_sizes(&col_tracks, content_width);
        let row_heights = Self::resolve_row_heights(container, &row_tracks, &col_widths);

        // 逐行列排列 GridItem
        for (i, child) in container.children.iter_mut().enumerate() {
            let col = i % cols;
            let row = i / cols;
            child.rect.x = start_x + col_widths[..col].iter().sum::<f32>();
            child.rect.y = start_y + row_heights[..row].iter().sum::<f32>();
        }
    }
}
```

支持轨道类型：`Fixed(px)`, `Flex(fr)`, `Percent(%)`, `Auto`。

## Blink 的实现

### LayoutNG：不可变 Fragment 输出 + 算法管道

LayoutNG 是 Blink 的新一代布局引擎（2019 年起逐步替代旧 Layout），核心设计：

1. **输入** = `NGConstraintSpace`（可用空间 + 约束条件）
2. **算法** = `NGLayoutAlgorithm`（各布局模式子类）
3. **输出** = `NGPhysicalBoxFragment`（不可变的布局结果）

```cpp
// ===== 约束空间：算法输入 =====
class NGConstraintSpace {
public:
    // 可用空间
    LogicalSize AvailableSize() const;
    // 是否固定宽度/高度
    bool IsFixedBlockSize() const;
    // 百分比解析基准
    LogicalSize PercentageResolutionSize() const;
    // 是否为初始/最终列
    bool IsInitialColumnBalancingPass() const;
    // BFC 偏移
    NGBfcOffset BfcOffset() const;
    // ...
};

// ===== 算法接口 =====
class NGLayoutAlgorithm {
public:
    // 核心方法：给定约束空间，计算布局结果
    virtual scoped_refptr<const NGLayoutResult> Layout() = 0;

    // 约束空间（输入）
    const NGConstraintSpace& ConstraintSpace() const;
    // 节点（DOM 或匿名框）
    NGBlockNode Node() const;
};

// ===== 布局结果 =====
class NGLayoutResult {
public:
    // 物理片段（布局输出）
    const NGPhysicalBoxFragment& PhysicalFragment() const;
    // 块级布局特有：子节点链表的根
    const NGBlockBreakToken* BlockBreakToken() const;
    // ...
};

// ===== 物理片段：不可变布局输出 =====
class NGPhysicalBoxFragment : public NGPhysicalFragment {
public:
    // 子片段
    const ChildrenVector& Children() const;
    // 尺寸
    PhysicalSize Size() const;
    // 溢出
    PhysicalRect ScrollableOverflow() const;
};
```

### NGBlockLayoutAlgorithm：块级布局

```cpp
class NGBlockLayoutAlgorithm : public NGLayoutAlgorithm {
public:
    scoped_refptr<const NGLayoutResult> Layout() override {
        // 1. 处理之前未完成的子节点（分页/分列断裂恢复）
        HandleSubtreeLayout();

        // 2. 遍历子节点，逐个布局
        for (NGBlockNode child = Node().FirstChild(); child; child = child.NextSibling()) {
            // 2a. 为子节点构建约束空间
            NGConstraintSpace child_space = CreateConstraintSpaceForChild(child, ...);

            // 2b. 调用子节点的 Layout()（多态）
            scoped_refptr<const NGLayoutResult> child_result = child.Layout(child_space, break_token);

            // 2c. 对结果进行 BFC 定位
            PlaceChildInBfc(child, child_result, &bfc_offset, ...);

            // 2d. 传播浮动信息
            if (child.IsFloating())
                HandleFloat(child, child_result);

            // 2e. 收集子片段
            container_builder.AddChild(child_result->PhysicalFragment(), offset);
        }

        // 3. 计算容器最终尺寸（考虑边距、浮动、BFC 规则）
        ComputeBlockSize(container_builder);

        // 4. 生成不可变片段
        return container_builder.ToBoxFragment();
    }
};
```

### NGFlexLayoutAlgorithm：Flexbox 布局

```cpp
class NGFlexLayoutAlgorithm : public NGLayoutAlgorithm {
    scoped_refptr<const NGLayoutResult> Layout() override {
        // 1. 收集 flex items（包括匿名框）
        HeapVector<NGFlexItem> flex_items;
        CollectFlexItems(&flex_items);

        // 2. 确定主轴尺寸（可用空间 → 轨道总宽/高）
        LayoutUnit main_axis_available = ComputeAvailableMainAxisSpace();

        // 3. 确定 flex-basis 并分配弹性空间（flex-grow/shrink）
        ResolveFlexibleLengths(&flex_items, main_axis_available);

        // 4. 交叉轴对齐（align-items / align-self / align-content）
        AlignItemsOnCrossAxis(&flex_items);

        // 5. 生成片段
        return container_builder.ToBoxFragment();
    }
};
```

### LayoutNG 与旧 Layout 的集成

```cpp
// LayoutBlockFlow::UpdateLayout() —— LayoutNG 入口
void LayoutBlockFlow::UpdateLayout() {
    // 1. 从 DOM/样式构建 NGBlockNode（输入）
    NGBlockNode block_node(this);

    // 2. 构建约束空间
    NGConstraintSpace constraint_space = BuildConstraintSpace(block_node);

    // 3. 选择算法并执行
    scoped_refptr<const NGLayoutResult> result;
    if (IsFlexibleBox()) {
        result = NGFlexLayoutAlgorithm(block_node, constraint_space).Layout();
    } else if (IsGrid()) {
        result = NGGridLayoutAlgorithm(block_node, constraint_space).Layout();
    } else {
        result = NGBlockLayoutAlgorithm(block_node, constraint_space).Layout();
    }

    // 4. 将片段结果应用到 LayoutObject
    const NGPhysicalBoxFragment& fragment = result->PhysicalFragment();
    ApplyFragmentToLayoutObject(fragment);
}
```

## 优劣势分析

| 维度 | 我们 | Blink LayoutNG |
|------|------|---------------|
| **Block 布局** | 手写 86 行 Y 堆叠 | NGBlockLayoutAlgorithm ~2000 行 |
| **Flexbox** | taffy 委托（Rust 社区 crate） | 自研 NGFlexLayoutAlgorithm |
| **Grid** | 手写 200 行基础实现 | 自研 NGGridLayoutAlgorithm（完整 Level 1） |
| **文本测量** | rustybuzz (HarfBuzz) + fontdb | HarfBuzz 直接调用 + 自研塑形缓存 |
| **约束传递** | 隐式（rect.width + viewport） | 显式 NGConstraintSpace 对象 |
| **布局输出** | 原地修改 LayoutBox.rect | 不可变 NGPhysicalBoxFragment |
| **算法选择** | `match box_type` 手动分发 | `NGBlockNode::Layout()` 多态 + 算法工厂 |
| **缓存** | 无 | (节点, 约束空间) → Fragment 缓存 |
| **外边距折叠** | 仅相邻兄弟折叠（基本） | 完整 BFC 规则（父子间、空块、负 margin） |
| **分页/多列** | 不支持 | 通过 BreakToken 支持分页 |

### 我们的优势

1. **Block 布局极简**：86 行代码覆盖 80% 场景，适合原型和学习
2. **taffy 生态复用**：Flexbox 实现由社区维护，跟随规范更新
3. **代码总量小**：全部布局算法 ~1000 行，Blink 仅 NGFlexLayoutAlgorithm 就 3000+ 行
4. **原地修改**：直接修改 LayoutBox.rect，无中间分配开销

### 我们的劣势

1. **缺少 BFC 隔离**：`overflow: hidden` 不能阻止 margin 穿透，浮动无法正确清除
2. **外边距折叠不完整**：仅处理同级兄弟，未处理父子折叠、空块折叠
3. **无布局缓存**：即使只有 1 个节点变了，整个树重算
4. **taffy 清空重建**：每次 flex 布局 `taffy_tree.clear()` + 重建所有节点，无法复用
5. **Grid 仅基础**：不支持 `grid-area`、`grid-template-areas`、`auto-fill`/`auto-fit`
6. **无百分比解析基准**：`width: 50%` 依赖构建阶段父容器宽度估计，不精确
7. **无 min-content / max-content**：固有尺寸关键字完全缺失
8. **无 aspect-ratio 替换元素**：`<img>` 的固有尺寸未处理

### Blink LayoutNG 的优势

1. **完整规范覆盖**：CSS 2.1 + Flexbox + Grid 完整实现，经过 WPT 测试套件验证
2. **不可变 Fragment 输出**：天然适合缓存 + 增量更新，布局输入不变直接复用
3. **显式约束空间**：所有约束参数化为 `NGConstraintSpace`，可独立测试
4. **BFC 完整实现**：外边距折叠、浮动清除、BFC 隔离全部符合规范
5. **BreakToken 分页**：支持打印/多列场景的跨页断裂
6. **百分比解析精确**：所有百分比值在正确基准下解析

### Blink LayoutNG 的劣势

1. **代码量巨大**：仅布局引擎核心代码数万行
2. **概念抽象多**：NGConstraintSpace / NGLayoutResult / NGFragment / BreakToken 概念负担
3. **Fragment 分配开销**：每次 Layout 产出一个新的堆分配 Fragment 对象

## 改进方向

### 短期

1. **完善 BFC 规则**：为 `overflow: hidden/scroll/auto` 的容器隔离外边距折叠
2. **Fragment 缓存**：为不变子树缓存 `(LayoutBox*, computed_style_hash, constraint) → (rect, children_offsets)` 元组
3. **百分比解析基准**：在递归时显式传递百分比基准尺寸，不再依赖构建阶段估计值

### 中期

4. **约束空间对象**：将隐式参数 `(viewport, parent_box_type, constrain_width)` 封装为 `ConstraintSpace`
   ```rust
   struct ConstraintSpace {
       available_size: Size<f32>,
       percentage_basis: Size<f32>,
       bfc_offset: Option<f32>,
       is_fixed_height: bool,
   }
   ```
5. **布局算法 trait**：统一算法接口，支持多态
   ```rust
   trait LayoutAlgorithm {
       fn layout(&self, node: &LayoutBox, space: &ConstraintSpace) -> Fragment;
   }
   ```

### 长期
